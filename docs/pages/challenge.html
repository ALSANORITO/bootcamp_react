Coding Challenge: Post It Notes App

Requirements

General:

  • The application's code is saved and documented in a private GitHub repository.
  • The repository contains a custom README.md file that specifies how to start the application locally, and documents any relevant technical decision made.
  • All project dependencies have a documented reason why they were added in the README.md file.

React:

  • The application follows the principles shown in class.
  • Base application is created using create-react-app.
  • React application routing is managed with react-router-dom.
  • Post it's state is saved in LocalStorage.
  • All React components follow SRP (Single Responsability Principle) by being small and reusable.
  • There's a clear file structure for the project and the React components.
  • The application follows airbnb style guide and there's a script to check all files automatically using eslint.
  • All visible React components have a css modules file if needed.
  • All React components follow the rules of hooks.
  • Semantic HTML is written when possible.

Features

The following features are mandatory, and expected to be developed with some styling:

  • As a user, I can create a new Post it.
  • As a user, I can see all Post its in the workspace.
  • As a user, I can edit a Post it.
  • As a user, I can move a Post it to the trash bin.
  • As a user, I can open the trash bin to see all Post its that have been moved there.
  • As a user, I can move a Post it from the trash bin to the workspace.
  • As a user, I can permanently delete all Post its in the trash bin.
  • As a user, I can see if the trash bin has Post its by looking the trash bin icon.
  • As a user, all Post its in the workspace and trash bin are saved in LocalStorage.
